## Введение

**AISPC** (Automated Information System for PC Assembly) — это современная ERP-система для автоматизации процессов сборки персональных компьютеров. Система предназначена для:

- Управления заказами клиентов
- Ведения базы клиентов и партнёров
- Контроля складских запасов компонентов
- Планирования и отслеживания сборочных заданий
- Формирования отчётности и аналитики

### Основные возможности

- 📋 **Управление заказами** — создание, редактирование, отслеживание статусов заказов
- 👥 **Клиентская база** — хранение информации о клиентах, история заказов
- 🛠️ **Каталог компонентов** — учёт комплектующих, цен, остатков на складе
- 📦 **Складской учёт** — поступление, списание, перемещение, инвентаризация
- 🔨 **Сборочные задания** — назначение задач сборщикам, контроль выполнения
- 📊 **Отчётность** — аналитика продаж, складских остатков, эффективности

## Начало работы

### Системные требования

- Операционная система: Windows 10/11
- Минимальное разрешение экрана: 1280×800
- Рекомендуемое разрешение: 1920×1080 и выше
- Подключение к базе данных (настраивается администратором)

### Запуск приложения

1. Запустите исполняемый файл `AISPC.exe`
2. Дождитесь загрузки окна авторизации

### Вход в систему

Окно авторизации разделено на две части:

**Левая панель — Форма входа:**
- Поле «Логин» — введите ваше имя пользователя
- Поле «Пароль» — введите пароль
- Кнопка «🚀 Войти в систему» — выполните вход
- Сообщения об ошибках отображаются под кнопкой входа

**Правая панель — Информация о системе:**
- Список основных возможностей системы
- Краткое описание каждого модуля

> **Примечание:** При первом входе обратитесь к администратору для получения учётных данных.

## Работа с заказами

### Просмотр заказов

1. В боковом меню выберите **«Управление заказами»**
2. Отобразится таблица всех заказов со следующими колонками:
   - № Заказа
   - Клиент
   - Статус
   - Сумма
   - Дата заказа
   - Приоритет
   - Примечания

### Создание нового заказа

1. Нажмите кнопку **«➕ Новый заказ»**
2. Откроется диалоговое окно создания заказа

**Заполнение информации о клиенте:**
- **Клиент** — название компании или ФИО
- **Приоритет** — выберите из списка (Низкий, Обычный, Высокий, Критический)
- **Примечания** — дополнительная информация о заказе

**Заполнение деталей заказа:**
- **Сумма заказа** — общая стоимость
- **Дата заказа** — дата оформления (по умолчанию текущая)

3. Проверьте предварительный просмотр внизу окна
4. Нажмите **«💾 Создать заказ»** или **«❌ Отмена»**

### Изменение статуса заказа

1. Выделите заказ в таблице
2. Нажмите кнопку **«✏️ Изменить статус»**
3. Выберите новый статус в диалоговом окне
4. Подтвердите изменение

### Просмотр деталей заказа

1. Выделите заказ в таблице
2. Нажмите кнопку **«📋 Детали заказа»**
3. Откроется окно с полной информацией о заказе, включая:
   - Состав заказа (компоненты)
   - История изменений статуса
   - Привязанное сборочное задание

## Управление клиентами

### Просмотр клиентской базы

1. В боковом меню выберите **«Клиенты»**
2. Отобразится таблица клиентов с колонками:
   - ID
   - Название/ФИО
   - Тип (ООО, ИП, Физическое лицо)
   - Телефон
   - Email
   - ИНН
   - Дата создания
   - Примечания

### Поиск клиентов

- Введите текст в поле поиска в верхней части страницы
- Нажмите **«🔍 Найти»** или используйте автопоиск при вводе
- Поиск осуществляется по названию, ФИО, телефону, email и ИНН

### Добавление нового клиента

**Способ 1: Через панель управления**
1. На панели управления нажмите **«👤 Новый клиент»**

**Способ 2: Через страницу клиентов**
1. На странице «Клиенты» нажмите **«➕ Добавить»**

**Заполнение формы:**

*Основная информация:*
- **Название/ФИО** — полное наименование или фамилия клиента
- **Тип клиента** — выберите из списка (ООО, ИП, Физическое лицо)
- **ИНН** — идентификационный номер налогоплательщика (необязательно)

*Контактная информация:*
- **Телефон** — контактный номер
- **Email** — адрес электронной почты
- **Адрес** — юридический или фактический адрес

*Примечания:*
- Дополнительная информация о клиенте

3. Проверьте предварительный просмотр
4. Нажмите **«💾 Добавить клиента»**

### Редактирование клиента

1. Выделите клиента в таблице
2. Нажмите кнопку **«✏️ Изменить»**
3. Внесите необходимые изменения в форме
4. Нажмите **«💾 Сохранить»**

### Просмотр заказов клиента

1. Выделите клиента в таблице
2. Нажмите кнопку **«📋 Заказы»**
3. Откроется список всех заказов выбранного клиента

## Каталог компонентов

### Просмотр компонентов

1. В боковом меню выберите **«Каталог компонентов»**
2. Отобразится таблица компонентов:
   - ID
   - Название
   - Категория
   - Бренд
   - Модель
   - Цена
   - На складе (количество)
   - Мин. остаток
   - Активен (статус)

### Поиск компонентов

- Введите название, модель или бренд в поле поиска
- Нажмите **«🔍 Найти»**
- Поиск работает по всем текстовым полям компонента

### Добавление компонента

1. Нажмите кнопку **«➕ Добавить»**
2. Заполните форму нового компонента:

*Основная информация:*
- **Название** — наименование компонента
- **Категория** — тип компонента (Процессор, Видеокарта, ОЗУ и т.д.)
- **Бренд** — производитель
- **Модель** — конкретная модель

*Ценовые параметры:*
- **Цена закупки** — стоимость приобретения
- **Цена продажи** — рекомендованная цена для клиента

*Складские параметры:*
- **Количество на складе** — начальный остаток
- **Минимальный остаток** — порог для уведомления о пополнении

*Дополнительно:*
- **Описание** — характеристики компонента
- **Активен** — флаг видимости в каталоге

3. Нажмите **«💾 Добавить компонент»**

### Редактирование компонента

1. Выделите компонент в таблице
2. Нажмите кнопку **«✏️ Изменить»**
3. Внесите изменения
4. Нажмите **«💾 Сохранить»**

### Удаление компонента

> **Внимание!** Удаление возможно только для компонентов, не используемых в активных заказах.

1. Выделите компонент в таблице
2. Нажмите кнопку **«🗑️ Удалить»**
3. Подтвердите удаление в диалоговом окне

## Складской учёт

### Просмотр складских остатков

1. В боковом меню выберите **«Управление складом»**
2. Отобразится таблица остатков:
   - Компонент
   - Категория
   - Остаток (текущее количество)
   - Мин. остаток
   - Статус (Норма, Мало, Критично)
   - Последнее движение (дата)

### Складские операции

#### Поступление товаров

1. Нажмите кнопку **«📦 Поступление»**
2. В диалоговом окне укажите:
   - Компонент (выбор из каталога)
   - Количество
   - Дата поступления
   - Поставщик (опционально)
   - Комментарий
3. Нажмите **«💾 Провести поступление»**

#### Списание товаров

1. Нажмите кнопку **«📤 Списание»**
2. В диалоговом окне укажите:
   - Компонент
   - Количество
   - Причина списания (Брак, Потеря, Использование и т.д.)
   - Комментарий
3. Нажмите **«💾 Провести списание»**

#### Перемещение между складами

> Примечание: Доступно при наличии нескольких складов в системе.

1. Нажмите кнопку **«🔄 Перемещение»**
2. Укажите:
   - Компонент
   - Склад источник
   - Склад назначение
   - Количество
   - Комментарий
3. Нажмите **«💾 Провести перемещение»**

#### Инвентаризация

1. Нажмите кнопку **«📋 Инвентаризация»**
2. Система сформирует ведомость текущих остатков
3. Внесите фактические количества
4. Система автоматически создаст документы излишков/недостач

### Статусы складских остатков

- ✅ **Норма** — остаток выше минимального уровня
- ⚠️ **Мало** — остаток接近 минимального уровня
- ❌ **Критично** — остаток ниже минимального уровня, требуется пополнение

## Управление сборкой

### Просмотр сборочных заданий

1. В боковом меню выберите **«Сборочные задания»**
2. Отобразится таблица заданий:
   - № Задачи
   - Заказ (связанный заказ)
   - Сборщик (ответственный сотрудник)
   - Статус
   - Дата начала
   - Примечания

### Создание новой задачи сборки

1. Нажмите кнопку **«➕ Новая задача»**
2. Заполните форму:
   - **Заказ** — выберите заказ из списка
   - **Сборщик** — назначьте ответственного сотрудника
   - **Приоритет** — установите приоритет задачи
   - **Плановая дата завершения** — срок выполнения
   - **Комментарий** — особые указания сборщику
3. Нажмите **«💾 Создать задачу»**

### Изменение статуса задачи

Доступные статусы:
- 🆕 **Новая** — задача создана, ожидает начала
- 🔧 **В работе** — сборщик приступил к выполнению
- ⏸️ **На паузе** — временно приостановлена (ожидание компонентов)
- ✅ **Завершена** — сборка выполнена успешно
- ❌ **Отменена** — задача отменена

1. Выделите задачу в таблице
2. Нажмите **«✏️ Изменить статус»**
3. Выберите новый статус
4. При необходимости добавьте комментарий
5. Подтвердите изменение

### Просмотр деталей задачи

1. Выделите задачу
2. Нажмите **«📋 Детали»**
3. Откроется информация:
   - Состав конфигурации ПК
   - Этапы сборки
   - Использованные компоненты
   - Время выполнения
   - Комментарии сборщика

## Администрирование

### Управление пользователями

**Назначение:** Создание и настройка учётных записей сотрудников.

1. В боковом меню выберите **«Пользователи»**
2. Таблица пользователей содержит:
   - ID
   - Логин
   - ФИО
   - Роль
   - Email
   - Дата создания
   - Активен

#### Добавление пользователя

1. Нажмите **«➕ Добавить пользователя»**
2. Заполните форму:
   - **Логин** — уникальное имя для входа
   - **Пароль** — временный пароль (пользователь сможет изменить)
   - **ФИО** — полное имя сотрудника
   - **Роль** — уровень доступа (Администратор, Менеджер, Сборщик, Кладовщик)
   - **Email** — для уведомлений
3. Нажмите **«💾 Создать пользователя»**

#### Роли пользователей

| Роль | Права доступа |
|------|--------------|
| **Администратор** | Полный доступ ко всем модулям |
| **Менеджер** | Заказы, клиенты, отчёты |

| **Кладовщик** | Складские операции, остатки |

#### Редактирование пользователя

1. Выделите пользователя
2. Нажмите **«✏️ Изменить»**
3. Внесите изменения (кроме логина)
4. Нажмите **«💾 Сохранить»**

#### Блокировка пользователя

1. Выделите пользователя
2. Нажмите кнопку блокировки
3. Пользователь не сможет войти в систему

### Управление сотрудниками

**Назначение:** Кадровый учёт сотрудников компании.

1. В боковом меню выберите **«Сотрудники»**
2. Таблица сотрудников:
   - ID
   - ФИО
   - Должность
   - Телефон
   - Email
   - Дата приёма
   - Зарплата
   - Активен

1. Нажмите **«➕ Добавить»**
2. Заполните анкету:
   - **ФИО** — полностью
   - **Должность** — из справочника должностей
   - **Дата приёма** — дата трудоустройства
   - **Контакты** — телефон, email
   - **Зарплата** — оклад или ставка
3. Нажмите **«💾 Сохранить»**

#### Карточка сотрудника

1. Выделите сотрудника
2. Нажмите **«📋 Карточка»**
3. Откроется полная информация:
   - Личные данные

   - Выполненные сборочные задания
   - Начисления зарплаты

## Отчёты и аналитика

### Формирование отчётов

1. В боковом меню выберите **«Отчеты»**
2. Выберите тип отчёта из списка

### Виды отчётов

#### Отчёт по продажам

- Период (дата начала и окончания)
- Группировка (по дням, неделям, месяцам)
- Фильтры (клиент, менеджер, статус)
- Показатели: количество заказов, сумма, средний чек

#### Отчёт по складу

- Текущие остатки по категориям
- Движение товаров за период
- Товары с критическим остатком
- Оборачиваемость запасов

#### Отчёт по сборке

- Производительность сборщиков
- Среднее время сборки
- Статусы задач за период
- Процент успешных сборок

#### Отчёт по клиентам

- Новые клиенты за период
- Активность клиентов (количество заказов)
- Топ клиентов по сумме заказов

### Экспорт отчётов

## Часто задаваемые вопросы

### ❓ Как восстановить забытый пароль?

Обратитесь к администратору системы для сброса пароля.

### ❓ Можно ли изменить созданный заказ?

Да, если заказ находится в статусе «Новый». Для заказов в работе изменения ограничены.

### ❓ Как узнать, какие компоненты нужны для заказа?

В деталях заказа отображается конфигурация ПК со списком необходимых компонентов.

### ❓ Что происходит при достижении минимального остатка?

Система помечает компонент статусом «Мало» или «Критично». Рекомендуется настроить уведомления.

### ❓ Как отменить сборочное задание?

Измените статус задачи на «Отменена» с указанием причины отмены.

### ❓ Можно ли работать в системе нескольким пользователям одновременно?

Да, система поддерживает многопользовательский режим работы.

### ❓ Где хранятся данные системы?

Данные хранятся в централизованной базе данных на сервере компании.
