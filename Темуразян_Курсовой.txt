ВВЕДЕНИЕ

В условиях цифровой трансформации вопросы эффективного управления ресурсами становятся критически важными. Рынок сборки ПК связан с обработкой больших объемов информации: от учета комплектующих до управления заказами.

Актуальность обусловлена тем, что ручной учет не отвечает современным требованиям. Отсутствие единой системы ведет к ошибкам при комплектации, потере данных о складских остатках и срыву сроков. Создание АИС позволяет автоматизировать рутинные операции.

Программное обеспечение «AISPC» (Automated Information System for PC Assembly) — ERP-решение для автоматизации бизнес-процессов компании по сборке компьютеров через централизованный учет компонентов, ведение базы клиентов и управление заказами.

Цель: проектирование, реализация и тестирование системы для автоматизации управления сборкой ПК.

Задачи:
• Изучение предметной области
• Формирование требований
• Проектирование базы данных
• Разработка на C# и WPF
• Реализация алгоритмов фильтрации и отчетности

Объект: процесс управления сборкой ПК.
Предмет: методы разработки информационных систем на основе реляционных БД и ООП.

1 ОБЩАЯ ЧАСТЬ

Описание предметной области

Предметная область — деятельность компании по сборке ПК на заказ. Управление включает мониторинг складских запасов, ведение клиентской базы и контроль заказов.

Ключевые характеристики:
• Каталог компонентов: наименование, категория, производитель, модель, характеристики
• Складской учет: количество, статус («в наличии», «резерв»)
• Управление заказами: конфигурация, стоимость, сроки, статус

AS-IS (текущее состояние):
• Менеджер принимает заказ в бумажный блокнот
• Данные заносятся в Excel вручную
• Информация передается сборщику с задержкой
• Контроль склада — ручная проверка журналов
• Статус отслеживается через звонки

TO-BE (проектируемое):
АИС «AISPC» объединяет этапы в единую цепочку. Однократный ввод данных. Автоматическая проверка наличия компонентов. Уведомления сборщику в реальном времени.

Обоснование автоматизации

Необходимость автоматизации продиктована системными проблемами:

1. Нарушение актуальности информации — остатки обновляются с задержкой, возникают конфликты с клиентами.

2. Фрагментарность данных о клиентах — дублирование записей, накопление ошибок, невозможность анализа истории.

3. Операционные риски — изменение статуса обнаруживается с опозданием.

Технико-экономическое обоснование: автоматизация сокращает издержки, исключает человеческий фактор, гарантирует эффективность.

Функциональные требования

Ключевые пользователи: администратор, менеджер, сборщик, кладовщик.

Требования:
• Ввод и редактирование данных о компонентах
• Управление учетными записями (администратор)
• Ведение карточек клиентов с историей заказов
• Создание заказов с проверкой наличия компонентов
• Учет поступления/выбытия компонентов (кладовщик)
• Просмотр задач сборки с обновлением статуса (сборщик)

Выбор средств реализации

Язык: C# (.NET 8) — строгая типизация, управление памятью.
Интерфейс: WPF + XAML — адаптивность, единообразие.
СУБД: MySQL — отказоустойчивость, транзакционность.
ORM: Entity Framework Core — объектно-ориентированный доступ к данным.

Выбранные средства формируют надежный технологический фундамент.

2 ТЕХНОЛОГИЧЕСКАЯ ЧАСТЬ

Проектирование базы данных

Информационная модель «custom_pc_shop». База приведена к 3НФ.

Сущности:
• Roles — роли доступа
• Employees — персонал
• Clients — клиенты
• Users — пользователи
• Components — комплектующие
• WarehouseItems — складские остатки
• Orders — заказы
• OrderItems — позиции заказов
• AssemblyTasks — задачи сборки

Связи реализованы через внешние ключи.

Описание таблиц

Roles — справочник ролей (Администратор, Менеджер, Сборщик, Кладовщик)

Employees — ФИО, должность, контакты, оклад

Clients — тип, контакты, ИНН, история

Users — логины, пароли, роли

Components — название, категория, бренд, цена, описание

WarehouseItems — количество, мин.уровень, статус

Orders — клиент, статус, сумма, даты, приоритет

OrderItems — заказ, компонент, количество, цена

AssemblyTasks — заказ, сборщик, статус, даты

Программная реализация

OrdersPage — фильтрация по статусу, приоритету, клиенту

ClientsPage — валидация телефона, email, ИНН (Regex)

ComponentsPage — поиск по названию, категории, бренду

WarehousePage — мониторинг остатков, уведомления

AssemblyTasksPage — обновление статуса, фиксация дат

Обработка исключений

Уровни защиты:
1. Интерфейс — мягкая валидация (PreviewTextInput, Regex)
2. Бизнес-логика — try-catch-finally для SQL
3. Целостность — транзакции
4. Уведомления — анализ ошибок, подсказки
